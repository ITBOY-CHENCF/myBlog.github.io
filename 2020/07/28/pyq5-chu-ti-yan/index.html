前言

这是一篇关于之前自己对运维工具化开发的一些分享，其实自己对运维的线路理解的一直很浅，直到自己一步一步接触，逐步发挥一些自己的长处通过编写工具、编写脚本来提高日常运维管理的效率，开始自学Python语言，其实也不算学习，学习如何配置编译环境后，就直接上手开撸了，直蹦实现运维工具目的发出！

主要技术

  • Python 2.7

    2.x 版本官方已经不再维护了，目前3.x 已经支持许多高级特性，建议大家如果真的开始接触Python的话还是直接上3.X版本，Python对运维工作者的友好性，我就不多说了，你懂的~

  • Pyqt5

    Python的图形化编程可利用的GUI 库多种多样，其中包括：Tkinter、wxPython、Pywin等，我曾经尝试过使用Tkinter，但是后来才发现Tkinter不持多线程，GUI的使用会导致页面卡死等待线程处理结束才能有响应，坑~~，转而开始使用了Pyqt5，Pyqt5是一套Python绑定Digia QT5应用的框架。它可用于2.x 和3.x 版本，主要借助事件、信号槽能够完成UI的动作指令，具有支持多线程、界面编辑可塑性强的特性。

  • Paramiko

    paramiko是用python语言写的一个模块，遵循SSH2协议，支持以加密和认证的方式，进行远程服务器的连接，包括命令执行、文件上传、下载，这也是制作Linux图形化运维工具最基本功能库。

  • Linux Shell

    工具目的是为解决手工维护命令执行，掌握基本的命令执行、Shell编写就能够制作一些实用的工具

    基本命令包括netstate、ps、top、grep等等

界面制作

通过QtDesigner设计界面，QtDesigner的设计符合MVC的架构，实现了视图和逻辑的分离，从而实现了开发的便捷，虽然不像Web前端那样网上很多实用的开源框架可以套用，但是利用QtDesigner的简单布局，制作工具的基本UI是没有太大问题的，想要设计更加美观的样式还是需要多拿捏一些（我想要的只要不要太丑就行。。）

*很关键的一点是QtDesigner设计的形成是以xxxx.ui形式的xml文件，不能够直接用于代码开发，但是能够通过转化将UI文件转化为.py文件用于工程的引用，为了方便起见我也直接把QtDesigner、pyuic都配置到Pycharm中去转化文件直接引用都非常快捷。

转化执行的命令如下：

pyuic5 -o xx.py xxx.ui    #转化ui为py文件

物理状态

包括系统基本参数、磁盘空间情况、性能情况、CUP占用进程监控，这些也是平时关注比较多的点，非实时刷新的信息状态。

这一块对熟悉Linux基本命令也是非常有帮助的，熟悉了这些基本命令，操作系统基本信息获取也就轻而易举了。

简单实现代码如下：

class UpdateStatusInfo(QThread):   #同样主要通过paramiko进行命令执行回显需要信息参数
  ip_date = pyqtSignal(str,str)
  def run(self):
    # CUP使用率
    main_W.cup_used_lab_val.setText(
        ssh_server("top -b -n 2 -d 0 | awk 'NR==3{print $5}'| awk -F \"%\" '{print 100-$1\"%\"}'", "show"))
    # 内存使用率
    main_W.mem_used_lab_val.setText(
      str(round(float(ssh_server("free -m | awk 'NR==2{print 100-($4+$6+$7)*100/$2}'", "show")), 1)) + "%")
    # 发行版本
    main_W.version_val_lab.setText(ssh_server("cat /etc/redhat-release 2>/dev/null", "show"))
    # 语言编码
    main_W.und_lab_val.setText(
      ssh_server("grep \"LANG=\" /etc/sysconfig/i18n | grep -v \"^#\" | awk -F '\"' '{print $2}'", "show"))
    # 最后启动
    main_W.lastup_lab_val.setText(ssh_server("who -b | awk '{print $3,$4}'", "show"))
    # 运行时间
    main_W.run_lab_val.setText(ssh_server("uptime | sed 's/.*up \([^,]*\), .*/\\1/'", "show"))
    # CUP TOP情况
    main_W.top_lab_val.setText(
      ssh_server("top b -n1 | head -18 | tail -13 | awk '{print $1,$2,$3,$9,$10,$11,$12,$13}'| column -t",
            "show_cup_top"))
    print_ts("系统信息加载完成")

服务状态

应用服务运行整体情况（应用服务启、停、服务状态、查看运行日志）

应用服务管理是我最初设想完成的目标，之前通过TKinker实现了基本功能但是由于UI假死的不友好性被我无情抛弃，Pyqt的信号槽机制非常灵活，简单的理解就是利用事件绑定传递信号，通过信号槽接收信号处理信息。应用列表内容目前也都是代码写死的，不过改成配置文件形式加载也并不是什么难事，这样做的好处也是能够具有更好的工具通用性即使在其他环境下也能够管理服务。

环境配置

针对不同类型应用服务环境部署进行一键式应用配置、应用数据库配置修改

我之前所接触的环境部署工作，主要是虚机化环境下通过环境克隆快速复制出新环境，再通过执行编写好的Shell脚本对相应的配置项进行更改，更改的内容主要是包括应用IP地址这类参数，那么通过工具能够更快速的完成这一系列步骤。（当然这里我也非常想了解圈内其他同僚们对于一些环境部署搭建都会采用怎么样方式进行高效便捷方式，欢迎留言交流~）

简单示例如下：

#远程上传Shell文件
def ssh_upload(fromPath,toPath):

    ssh = paramiko.SSHClient()
    ssh.set_missing_host_key_policy(paramiko.AutoAddPolicy())
    ssh.connect(hostname=ip, port=22, username=user, password=pwd)

    sftp_client=paramiko.SFTPClient.from_transport(ssh.get_transport())
    try:
        sftp_client.put(fromPath, toPath)
    except Exception,e:
        print str(e)
        reply = QMessageBox.question(main_W, '提示', "文件上传更新失败！！", QMessageBox.Ok)
       sftp_client.close()
    ssh.close()

数据库环境虚机克隆后往往都会出现一些失效对象，那么就是需要手动再进行一次编译，工具也集成了这样的功能，简单实现方式如下：

#ORACLE数据库 使用cx_Oracle模块用与oracle数据库连接
class InvalidCommpile(QThread):
    invalid_commpile_date=pyqtSignal(list,str)     #事件信号
    def run(self):
        invalid_object_sql='''
            select owner,object_name,replace(object_type,'','') object_type,status from all_objects where status='INVALID' 
            '''
        try:
            conn = cx_Oracle.connect("用户名", "密码", "连接串",mode=cx_Oracle.SYSDBA)   #连接数据
        except cx_Oracle.DatabaseError as e:
            print_ts("数据库连接异常")
        else:
            cur = conn.cursor()                   #数据游标
            cur.execute(invalid_object_sql)       #执行查询
            result = cur.fetchall()               #获取数据集
            for item in result:
                if item[2] in ['INDEX','VIEW','PACKAGE BODY','PROCEDURE','TRIGGER','FUNCTION','TYPE','PACKAGE']:
                    owner=str(item[0])
                    object_name=str(item[1])
                    object_type=str(item[2])
                    sql_val = "alter"+ " " +object_type.strip('BODY')+" "+owner+"."+object_name+" "+"compile"          
                    try:
                        cur.execute(sql_val)      #执行失效对象编译
                    except cx_Oracle.DatabaseError as e:
                        log_list.append(time_val + convert_val(str(e)))
            cur.close()
            conn.close()
